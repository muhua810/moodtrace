## 💡 项目背景

当代大学生面临学业、就业、人际关系等多重压力，心理健康问题日益受到关注。然而大多数心理自助工具门槛高、缺洞察、隐私堪忧。**心迹**用「一句话记录 + AI 自动分析 + 可视化洞察」的方式，让心理健康自我管理变得简单。

## ✨ 核心功能

- ✍️ **每日情绪记录** — 一句话描述今日感受，3 秒完成
- 🤖 **AI 情绪分析** — 四层降级策略，零配置即可使用，详见 [AI 分析架构](#-ai-分析架构)
- 📊 **情绪热力图日历** — 自研 SVG 组件，GitHub 贡献图风格，年/月双视图
- 📈 **多维度统计** — 饼图/趋势图/柱状图 + 年度报告 + 工作日/周末分析
- 💛 **渐进式关怀** — 3 级关怀系统 + 危机关键词检测 + 心理援助热线
- 🔒 **隐私优先** — 纯前端存储 + AES-256-GCM 加密，数据不出设备
- 🌐 **群体情绪统计** — 首页展示群体情绪分布，前后端协作一目了然
- 📱 **PWA 支持** — 离线缓存、可安装到手机桌面
- ☁️ **云端备份** — 基于设备 ID 的无注册备份/恢复
- 🌍 **多语言支持** — 中/英/日/韩/法/西 6 种语言，设置页一键切换

## 🧠 AI 分析架构

心迹的情绪分析采用 **四层降级策略**，确保在任何网络环境下都能给出分析结果：

```
用户输入一句话
       │
       ▼
┌─────────────────────┐
│ 第一层：Workers AI   │  ← Cloudflare 边缘节点代理调用 LLM，零配置可用
│ （云端大模型推理）    │     首次用户无需任何设置即可使用 AI 分析
└────────┬────────────┘
         │ 失败
         ▼
┌─────────────────────┐
│ 第二层：用户自定义 API│  ← 用户在设置中填入自己的 API Key + 模型
│ （OpenAI/其他兼容）   │     支持 GPT-3.5/4、Claude 等任何兼容接口
└────────┬────────────┘
         │ 失败
         ▼
┌─────────────────────┐
│ 第三层：关键词引擎    │  ← 纯本地，200+ 中英文关键词 + 否定词检测
│ （规则匹配 + 否定翻转）│    + 反讽检测 + emoji 识别 + 分句加权
└────────┬────────────┘
         │ 置信度 < 0.5
         ▼
┌─────────────────────┐
│ 第四层：统计分析器    │  ← TF-IDF + 朴素贝叶斯，内置 200 条标注语料
│ （机器学习分类）      │     支持增量学习（用户反馈后动态调整）
└─────────────────────┘
```

**关键技术点：**
- **Workers AI**：利用 Cloudflare Workers 作为代理，用户无需配置 API Key 即可使用 AI（Workers 免费额度每日 10,000 次调用）
- **sanitizeAiResult**：对 AI 返回值做严格校验和清洗，防止注入攻击和异常数据
- **关键词引擎**：分句 → 否定词检测 → 反讽检测 → emoji 加权 → 加权平均，覆盖 200+ 中文情绪表达
- **统计分析器**：字符 bigram 分词 → TF-IDF 特征提取 → 朴素贝叶斯分类，200 条训练语料

## 🔒 隐私架构

心迹的隐私设计遵循 **数据最小化** 原则：

| 数据类型 | 存储位置 | 是否上传云端 | 说明 |
|----------|----------|:----------:|------|
| 情绪记录 | 浏览器 localStorage | ❌ | 默认本地存储 |
| 加密后的记录 | 浏览器 localStorage | ❌ | AES-256-GCM 加密后仍存本地 |
| 云端备份 | Cloudflare KV | ✅（用户主动触发） | 基于设备 ID，无需注册，可随时删除 |
| 群体情绪统计 | Cloudflare KV | ✅（匿名聚合） | 仅上传情绪类别（1-5），不上传文字内容 |
| API Key | 浏览器 sessionStorage | ❌ | 仅会话期间保留，关闭浏览器即清除 |

**"群体情绪统计"说明：** 首页展示的群体分布数据是匿名聚合结果。用户提交记录时，仅将情绪分数（1-5）以匿名方式发送到后端进行统计，**不包含任何文字内容、设备 ID 或个人信息**。个人数据和群体统计完全隔离。

```bash
# 克隆项目
git clone https://github.com/muhua810/moodtrace.git
cd moodtrace

# 安装依赖
npm install

# 启动开发服务器
npm run dev        # → http://localhost:3000

# 运行测试
npm test                    # 一次性运行
```

## 📁 项目结构

```
moodtrace/
├── src/
│   ├── components/        # UI 组件（热力图、关怀卡片、关键词云等）
│   ├── pages/             # 页面（首页、记录、统计、设置）
│   ├── services/
│   │   ├── analyze/       # 分析子模块（关键词库、emoji映射、否定词、反讽检测）
│   │   ├── aiService.js   # AI 调用服务（Workers 代理 + 用户自定义 API）
│   │   ├── emotionAnalyzer.js  # 情绪分析主入口（四层降级编排）
│   │   ├── statisticalAnalyzer.js  # TF-IDF + 朴素贝叶斯统计分析器
│   │   ├── storage.js     # 本地存储（支持 AES-256-GCM 加密）
│   │   ├── backupService.js  # 云端备份/恢复
│   │   └── apiService.js  # API 通信
│   ├── utils/             # 工具函数（加密、情绪类型）
│   ├── i18n/              # 国际化（翻译字典 + 工具模块，6 种语言）
│   ├── hooks/             # 自定义 React Hooks
│   └── contexts/          # React Context（主题切换）
├── worker/                # Cloudflare Worker 后端
├── docs/                  # 项目文档 + SVG 架构图
└── public/                # PWA 静态资源 + 图标
```
